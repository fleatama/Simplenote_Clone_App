# 20. 一括削除機能の実装

大量のメモを効率よく整理するため、複数のメモを選択して一度に削除できる「一括削除（Bulk Delete）」機能を実装した。

## 主な作業内容

1.  **選択状態の管理 (State Management):**
    - `checkedIds` という新しい状態を追加。
    - 重複を許さず、検索も高速な `Set` データ構造 (`new Set<string>()`) を使用して、チェックされたメモのIDを管理するようにした。

2.  **UIの拡張:**
    - メモ一覧の各アイテムの左側にチェックボックスを配置。
    - サイドバーのヘッダーに「すべて選択/解除」用のチェックボックスを追加。
    - 1つ以上のメモが選択された時のみ表示される、赤い「一括削除（ゴミ箱）」ボタンを実装。

3.  **一括削除ロジック:**
    - `handleBulkDelete` 関数を実装。
    - `Promise.all` を活用し、複数の削除APIリクエストを並列で実行することで、処理の高速化を図った。
    - 削除完了後、`notes` 状態から一括で対象を除外し、`checkedIds` をリセットする一連のクリーンアップ処理を実装した。

## 習得した英単語
- **Bulk:** 一括、大量。
- **Toggle:** 状態の反転切り替え。
