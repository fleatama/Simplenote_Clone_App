# 17. Vitestによる自動テストの導入

アプリケーションの品質を担保するため、自動テストツールを導入し、最初の単体テストを実装した。

## 主な作業内容

1.  **テスト環境の構築:**
    - `Vitest`, `jsdom`, `@testing-library/react` をインストール。
    - OS 11 (Big Sur) 環境での esbuild 互換性問題を、特定バージョンの指定と `--legacy-peer-deps` の活用により解決。
    - `vitest.config.ts` および `vitest.setup.ts` を作成。

2.  **コードのリファクタリング:**
    - `app/page.tsx` 内に散らばっていた便利関数（タイトル生成、日時フォーマット）を、テストしやすいように `utils/helpers.ts` へ切り出し。

3.  **単体テストの実装:**
    - `utils/helpers.test.ts` を作成。
    - 文字列の切り出しロジック、改行の扱い、日時変換の各ケースについて、6つのテスト項目（Assertion）を記述。
    - `npx vitest run` を実行し、すべてのテストが合格することを確認。
