# 8. 自動保存(オートセーブ)機能の実装

ユーザーがメモの編集をやめてから数秒後に、変更内容を自動でサーバーに保存する機能を実装する。これにより、ユーザーは保存ボタンを意識することなく、シームレスに作業を継続できる。

## 実装予定の作業

1.  **変更の検知:**
    - `useEffect` を使い、編集エリアの内容 (`selectedNoteContent`) の変更を監視する。

2.  **デバウンス処理の実装:**
    - ユーザーがタイピングを終えてから一定時間（例: 2秒）が経過したことを検知するために、「デバウンス」というテクニックを `setTimeout` と `clearTimeout` を使って実装する。
    - これにより、一文字入力するたびに保存が走るのを防ぎ、サーバーへの負荷を軽減する。

3.  **APIの呼び出し:**
    - デバウンス処理の後、`/api/notes/[id]` エンドポイントに対して `PUT` リクエストを送信し、変更された内容をサーバーに保存する。

4.  **UIへのフィードバック:**
    - 保存中や保存完了をユーザーにさりげなく伝えるためのUI（例: 小さなスピナーや「保存済み」のテキスト）の表示を検討する。
