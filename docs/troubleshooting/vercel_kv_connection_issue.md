# Vercel KV 接続エラーと解決手順のナレッジ

Vercel KV (Upstash) 使用時に発生するデータベース接続エラーに対する、診断と解決のナレッジ集。

## 1. エラーの種類と原因
- **ENOTFOUND / ERR_INVALID_URL**:
    - **原因**: プロジェクトの環境変数 `KV_REST_API_URL` が正しく設定されていないか、古い情報（存在しないURL）を参照している。
    - **特徴**: ログに `/pipeline` への接続失敗や、存在しないホスト名が表示される。

## 2. 解決のための診断手順
1. **ログの確認**: `DEBUG` ログを仕込み、`Attempting to save to Redis key` の後のエラーを確認する。
2. **Vercel CLI の確認**: `vercel env pull .env.local` で現在Vercelが認識している環境変数を確認する。
3. **Storage 設定の確認**:
    - ダッシュボードの [Storage] タブで接続状態を確認する。
    - "Connect" 状態になっていない場合は再接続を行う。
    - 接続できてもエラーが出る場合は、一度 Remove して新しくデータベースを作り直す。

## 3. 推奨する作業フロー (エラー回避のために)
- **環境変数を手動でいじりすぎない**: Vercel KV を使っている場合は、可能な限り `[Storage]` ダッシュボードの接続機能に任せ、自動設定される環境変数を信頼する。
- **再デプロイを忘れない**: 環境変数を変更した後は、必ず [Deployments] から Redeploy を行うこと。
- **デバッグログを早期に仕込む**: ネットワークの問題なのか、認証の問題なのかを判断するために、まずは「入り口」でログを出す。
