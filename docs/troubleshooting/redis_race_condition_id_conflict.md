# 新規作成ボタン連打によるID衝突 (Race Condition)

## 現象
「新規作成」ボタンを短時間に連打すると、同じ内容のメモが複数作成されたり、エラーが発生したりする。

## 原因
ID生成に「現在の最大ID + 1」という数値の連番方式を採用していたため。
複数のリクエストが同時に「今の最大ID」を読み込んでしまい、同じ「次のID」を使おうとして衝突（Race Condition）が発生した。

## 解決策
ID生成を「数値の連番」から、**UUID (Universally Unique Identifier)** に変更した。
```typescript
const newId = crypto.randomUUID();
```
これにより、1秒間に何回実行されても、生成されるIDが世界中で唯一無二のものになり、衝突が根本的に解決された。
