# URL指定時のテンプレートリテラル(バッククォート)記述ミス

## 現象
APIを呼び出す際、URLにIDを埋め込んでいるはずが、IDが展開されずに `${noteId}` という文字がそのままURLとして送信され、404エラーになる。

## 原因
JavaScriptの変数を文字列に埋め込む際、囲み記号に **シングルクォート (`'`)** や **ダブルクォート (`"`)** を使っていたため。変数を展開するには **バッククォート (`` ` ``)** を使う必要がある。

## 解決策
文字列の囲み記号をバッククォートに変更した。
```typescript
// × 間違い
const url = '/api/notes/${noteId}';

// ○ 正解
const url = `/api/notes/${noteId}`;
```
これにより、`${noteId}` の部分が実際の値（例: `1`）に正しく置き換わるようになった。
