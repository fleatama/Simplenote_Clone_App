# Neovim which-key の設定読み込み順序エラー

## 現象
起動時に「Invalid (empty) LHS」や「wk is nil」といったエラーが発生し、キーマップが正しく登録されない。

## 原因
`which-key` プラグインが読み込まれる（セットアップされる）前に、キーマップを登録するファイル (`keymaps.lua`) が実行されてしまっていた。

## 解決策
`init.lua` 内のプラグイン設定の `config` 関数内で `require('keymaps')` を呼び出すように変更し、プラグインの準備が整ってから設定を読み込むという順序を徹底する。
